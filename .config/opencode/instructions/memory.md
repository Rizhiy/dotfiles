# Memory Usage

The `memory` tool is available through `opencode-mem`.

Before starting a new task, search memory for relevant user preferences, project context, setup details, or prior decisions. Use targeted queries based on the task, such as `dotfiles`, `opencode`, `terminal`, `chrome`, or the project/tool name.

Use stored memories to avoid repeating prior mistakes and to follow established user workflows. If the user asks to remember something durable, store it with appropriate tags and scope.

When the user corrects your behavior, assumptions, workflow, or implementation choices, consider whether the correction represents a durable preference or process rule. If it does, save it to memory automatically with concise content and relevant tags. Do not store one-off corrections, transient task details, or sensitive information unless the user explicitly asks.
